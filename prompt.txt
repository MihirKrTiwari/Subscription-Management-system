Build a complete Subscription Management System as a web application with the following strict requirements.

TECH STACK (MANDATORY):

* Frontend: HTML, CSS, JavaScript
* Backend: Java (Servlets or simple Java HTTP server)
* Database: CSV file (no SQL database)

PROJECT OBJECTIVE:
Manage user subscriptions with plan selection, upgrades, downgrades, and cancellations.

---

1. CORE FEATURES:

* Display subscription plans (Basic, Standard, Premium)
* Each plan must include:

  * Price
  * Duration
  * Features
* User can:

  * Subscribe to a plan
  * Upgrade plan
  * Downgrade plan
  * Cancel subscription
* Show current active subscription on dashboard

---

2. TECHNICAL REQUIREMENTS (VERY IMPORTANT):

* Use ARRAYS to store predefined plans in Java
* Use CLASSES (OOP):

  * User class
  * Subscription class
* Implement METHOD OVERLOADING in Java
* Use CONTROL STATEMENTS (if-else / switch) for:

  * Plan selection
  * Validation logic

---

3. CSV DATABASE REQUIREMENTS:

* Use a CSV file to store user data:
  Fields:
  userId, name, email, currentPlan, startDate, endDate
* Implement:

  * Read from CSV
  * Write to CSV
  * Update records
* Use Java file handling (BufferedReader, FileWriter)

---

4. BACKEND FUNCTIONALITY:

* Create APIs/endpoints:

  * Get all plans
  * Subscribe user
  * Update subscription
  * Cancel subscription
  * Get user details
* Handle GET and POST requests properly

---

5. FRONTEND REQUIREMENTS:

* Pages:

  * Home (plans display)
  * Dashboard
  * Subscription management
* Use fetch() to call Java backend APIs
* Dynamically update UI
* Show success/error messages

---

6. FILE STRUCTURE:

* frontend/

  * index.html
  * style.css
  * script.js
* backend/

  * Main.java / Servlets
  * User.java
  * Subscription.java

  * data.csv
